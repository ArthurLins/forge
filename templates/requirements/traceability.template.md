## How it works

1. Requirements are defined with stable IDs (`FR`, `NFR`, `BR`, `CR`, `UC`,
   `EN`, `ADR`) in the other documents — see the [index](index.md).

3. The tool scans the source and emits the matrix below, so every requirement's
   coverage (and any **gap** — a requirement with no code or no test) is visible
